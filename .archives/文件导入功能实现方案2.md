## 四、CacheableConverter 适配问题

你指出了关键问题：如果 `CacheableConverter` 同时实现 `ExportableConverter` 和 `ImportableConverter`，那么只支持导出的转换器（如 `WhenNullConverter`）就无法继承它了。

### 解决方案：拆分为两个缓存基类

```java
/**
 * 导出可缓存转换器
 */
public abstract class CacheableExportConverter implements ExportableConverter {
  private final Map<String, Object> cache = new ConcurrentHashMap<>();

  @Override
  public Object convertForExport(Object input, List<Object> arguments) {
    String cacheKey = key(input, arguments);
    return cache.computeIfAbsent(cacheKey, k -> doConvertForExport(input, arguments));
  }

  @Override
  public boolean supportsExport(Class<?> sourceType) { return true; }

  protected abstract Object doConvertForExport(Object input, List<Object> arguments);

  private String key(Object input, List<Object> arguments) {
    return Strings.value(input, "") + "#" + Digests.blake3(arguments);
  }
}

/**
 * 导入可缓存转换器
 */
public abstract class CacheableImportConverter implements ImportableConverter {
  private final Map<String, Object> cache = new ConcurrentHashMap<>();

  @Override
  public Object convertForImport(Object input, List<Object> arguments, Class<?> targetType) {
    String cacheKey = key(input, arguments) + "#" + targetType.getName();
    return cache.computeIfAbsent(cacheKey, k -> doConvertForImport(input, arguments, targetType));
  }

  @Override
  public boolean supportsImport(Class<?> targetType) { return true; }

  protected abstract Object doConvertForImport(Object input, List<Object> arguments, Class<?> targetType);

  private String key(Object input, List<Object> arguments) {
    return Strings.value(input, "") + "#" + Digests.blake3(arguments);
  }
}

/**
 * 双向可缓存转换器 — 同时支持导出和导入
 */
public abstract class CacheableConverter extends CacheableExportConverter implements ImportableConverter {
  private final Map<String, Object> importCache = new ConcurrentHashMap<>();

  @Override
  public Object convertForImport(Object input, List<Object> arguments, Class<?> targetType) {
    String cacheKey = key(input, arguments) + "#" + targetType.getName();
    return importCache.computeIfAbsent(cacheKey, k -> doConvertForImport(input, arguments, targetType));
  }

  @Override
  public boolean supportsImport(Class<?> targetType) { return true; }

  protected abstract Object doConvertForImport(Object input, List<Object> arguments, Class<?> targetType);

  private String key(Object input, List<Object> arguments) {
    return Strings.value(input, "") + "#" + Digests.blake3(arguments);
  }
}
```

### 各转换器的继承关系

| 转换器 | 继承 | 说明 |
|--------|------|------|
| WhenNullConverter | `CacheableExportConverter` | 仅导出 |
| WhenEmptyConverter | `CacheableExportConverter` | 仅导出 |
| DateConverter | `CacheableConverter` | 导出+导入 |
| NumberConverter | `CacheableConverter` | 导出+导入 |
| MappingConverter | `CacheableConverter` | 导出+导入 |

## 五、@DataValidator 增加 execution 参数

赞同。用注解声明执行时机，而非在执行器中硬编码特殊逻辑。

### 注解定义

```java
@Retention(RetentionPolicy.RUNTIME)
@Target(ElementType.TYPE)
public @interface DataValidator {
  /**
   * 校验器名称
   */
  String name();

  /**
   * 校验器执行优先级，数字越小优先级越高
   */
  int priority();

  /**
   * 执行时机
   */
  Execution execution() default Execution.IMMEDIATE;

  /**
   * 校验器执行时机
   */
  enum Execution {
    /**
     * 即时执行 — 在校验阶段逐行执行
     */
    IMMEDIATE,
    /**
     * 延后执行 — 在所有行处理完成后执行（如唯一性校验）
     */
    DEFERRED
  }
}
```

### 各校验器的注解配置

| 校验器 | name | priority | execution |
|--------|------|----------|-----------|
| RequiredValidator | `required` | 0 | IMMEDIATE |
| LengthValidator | `length` | 10 | IMMEDIATE |
| PatternValidator | `pattern` | 20 | IMMEDIATE |
| DependencyValidator | `dependency` | 40 | IMMEDIATE |
| RangeValidator | `range` | 100 | IMMEDIATE |
| UniqueValidator | `unique` | 30 | DEFERRED |

### ChainableValidatorExecutor 中的处理

```java
public class ChainableValidatorExecutor {
  private final List<Validator> immediateChain;   // IMMEDIATE 校验器
  private final List<Validator> deferredChain;     // DEFERRED 校验器

  // 逐行执行即时校验
  public List<ValidateResult> executeImmediate(Object input, ValidateContext context) { ... }

  // 全量数据后执行延后校验
  public List<ValidateResult> executeDeferred(Object input, ValidateContext context) { ... }
}
```

`ImportPipeline` 中的流程自然变为：

```
逐行处理：
  Cleanse → Validate(immediate) → Convert → PostValidate(immediate) → Construct

全量完成后：
  GlobalValidate(deferred) → UniqueValidator 等
```

无需特殊分支逻辑，扩展时只需在注解中声明 `execution = Execution.DEFERRED` 即可。
