```java
/**
 * 导出转换器：Bean字段值 → 展示文本
 */
public interface ExportableConverter extends Converter {
  /**
   * 执行导出转换
   * @param input Bean字段的原始值
   * @param arguments 转换参数（来自YAML配置）
   */
  Object convertForExport(Object input, List<Object> arguments) throws DataConvertException;

  /**
   * 判断是否能处理来自此Java类型的值
   */
  boolean supportsExport(Class<?> sourceType);
}

/**
 * 导入转换器：用户输入文本 → Bean字段值
 */
public interface ImportableConverter extends Converter {
  /**
   * 执行导入转换
   * @param input 用户输入的字符串值
   * @param arguments 转换参数（来自YAML配置）
   * @param targetType 目标Java类型（用于类型精确转换）
   */
  Object convertForImport(Object input, List<Object> arguments, Class<?> targetType) throws DataConvertException;

  /**
   * 判断是否能将值转换为此目标Java类型
   */
  boolean supportsImport(Class<?> targetType);
}
```

### 2.2 `convertForImport` 增加 `targetType` 参数的理由

导入时需要知道最终要转成什么类型。例如 `DateConverter` 需要知道目标是 `Date`、`LocalDate` 还是 `LocalDateTime`，才能决定用哪个解析器。导出时不需要，因为值本身已经是正确的类型。

### 2.3 各转换器实现调整

| 转换器 | ExportableConverter | ImportableConverter | 导入 convertForImport 逻辑 |
|--------|:---:|:---:|------|
| WhenNullConverter | ✅ | ❌ | — |
| WhenEmptyConverter | ✅ | ❌ | — |
| DateConverter | ✅ | ✅ | 用 `pattern.date` 解析字符串为目标日期类型 |
| NumberConverter | ✅ | ✅ | 用 `pattern.number` 解析字符串为目标数字类型 |
| MappingConverter | ✅ | ✅ | 反向映射：value → key |
| EnumConverter | ✅ | ✅ | 反向枚举：label → 枚举原始值 |
| DesensitizeConverter | ✅ | ❌ | — |
| TruncateConverter | ✅ | ❌ | — |
| BoolConverter | ✅ | ✅ | 反向布尔：文本 → "true"/"false" |
| PrefixConverter | ✅ | ❌ | — |
| SuffixConverter | ✅ | ❌ | — |
| UppercaseConverter | ❌ | ✅ | 字符串转大写 |

### 2.4 CacheableConverter 适配

```java
public abstract class CacheableConverter<O> implements ExportableConverter, ImportableConverter {
  // 导出缓存
  private final Map<String, O> exportCache = new ConcurrentHashMap<>();
  // 导入缓存
  private final Map<String, O> importCache = new ConcurrentHashMap<>();

  @Override
  public Object convertForExport(Object input, List<Object> arguments) {
    String cacheKey = key(input, arguments);
    return exportCache.computeIfAbsent(cacheKey, k -> doConvertForExport(input, arguments));
  }

  @Override
  public Object convertForImport(Object input, List<Object> arguments, Class<?> targetType) {
    String cacheKey = key(input, arguments) + "#" + targetType.getName();
    return importCache.computeIfAbsent(cacheKey, k -> doConvertForImport(input, arguments, targetType));
  }

  protected abstract O doConvertForExport(Object input, List<Object> arguments);
  protected abstract O doConvertForImport(Object input, List<Object> arguments, Class<?> targetType);

  // 未实现 ImportableConverter 的转换器，抛出 UnsupportedOperationException
  // 由子类选择性覆盖
}
```

```java
  public List<ValidateResult> execute(Object input, ValidateContext context) {
```

### 4.7 UniqueValidator 的特殊处理

`UniqueValidator` 无法在逐行处理时完成——需要收集所有行的值后才能判断重复。处理策略：

1. 逐行处理时不执行 `UniqueValidator`
2. 全部行处理完毕后，执行一次全局唯一性检查
3. 标记重复的所有行为失败，错误信息指明哪些行号与当前行重复

## 八、导入模板结构

### 8.1 Excel 模板布局

```
#列(序号列) | name        | phone       | status        | amount      | createTime
────────────────────────────────────────────────────────────────────────────────────
HEADER      | name        | phone       | status        | amount      | createTime        ← Row 1
HEADER      | 属性名称     | 电话        | 状态          | 金额        | 创建时间           ← Row 2
MOCK        | sampleName  | 13800138000 | 1             | 1000.00     | 2024-01-01        ← Row 3（可选）
1           |             |             |               |             |                    ← Row 4+（用户数据）
2           |             |             |               |             |                    ← Row 5+
```

### 8.2 各行详细规范

**Row 1 — 代码标识行**
- `#` 列：值 `HEADER`，背景色浅蓝，批注：「代码标识行，请勿修改，否则导入将失败」
- 数据列：值 = 字段名（columns的key），背景色浅蓝，批注内容包含该列的完整填写说明
- 批注内容格式：
  ```
  【{header}】
  类型：{Java类型}
  必填：是/否
  最大长度：{maxLength}字
  格式：{pattern描述}
  允许值：{mapping的key=value列表}
  示例：{mock数据}
  ```

**Row 2 — 人类阅读行 + 格式提示**
- `#` 列：值 `HEADER`，背景色浅蓝，批注：「列名称行，可阅读但请勿修改」
- 数据列：值 = i18n 解析后的 header 文本，背景色浅蓝
- 批注内容 = 格式提示（仅当 `with-hint-row: true` 时添加）：
  - 必填字段：`*必填`
  - 选填字段：`选填`
  - 长度限制：`≤{maxLength}字`
  - 正则格式：`{pattern的人类可读描述}`
  - 映射值：`1=启用/2=禁用`
  - 范围限制：`0~99999999.99`
  - 多个提示用 ` | ` 分隔，如：`*必填 | ≤50字`

**Row 3 — 示例数据行（可选）**
- `#` 列：值 `MOCK`，背景色浅黄，批注：「示例数据，仅供参考，导入时将自动忽略」
- 数据列：值 = mock.json 中对应的值，背景色浅黄
- 无 mock.json 文件则不生成此行

**Row 4+ — 用户数据区**
- `#` 列：值 = 序号（1, 2, 3...），背景色白色
- 数据列：空白，背景色白色

### 8.3 Excel 数据验证（Data Validation）

对于有 `mapping` 配置的列，在 Row 4+ 的单元格上设置下拉列表验证，选项为 mapping 的 values（如"启用"、"禁用"），从源头避免输入错误值。

### 8.4 导入时的行类型识别

通过 `#` 列的值判断行类型：

| `#` 列值 | 行类型 | 处理方式 |
|----------|--------|---------|
| `HEADER` | 表头行 | 跳过，用于列匹配 |
| `HINT` | 提示行 | 跳过（当前设计已合并到Row2批注，但保留此标记以备扩展） |
| `MOCK` | 示例行 | 跳过 |
| 数字（≥1） | 数据行 | 执行导入管线 |
| 空白 | 无标记数据行 | 执行导入管线（兼容无#列的简易模式） |

### 8.5 列匹配逻辑

1. 读取 Row 1（代码标识行），通过 `#` 列值 `HEADER` 确认
2. 从 Row 1 的数据列中提取字段名
3. 与 `DataDefinition.columns` 的 key 精确匹配
4. 列的位置无关——字段名匹配，而非列序号匹配

## 九、导入管线（ImportPipeline）

### 9.1 完整流程

```
文件输入
  │
  ├─ [Excel] → Fesod ReadListener 流式读取 → 逐行获取 字段名→字符串值 映射
  │
  ├─ [JSON]  → Jackson JsonParser 流式解析 → 逐对象获取 字段名→值 映射
  │
  ├─ [XML]   → StAX XMLStreamReader 流式解析 → 逐<record>获取 字段名→值 映射
  │
  └─ [YAML]  → SnakeYAML 逐文档解析 → 逐对象获取 字段名→值 映射
                                                │
                                                ▼
                                    ┌─ 识别行类型 ──────────┐
                                    │  通过 # 列判断         │
                                    │  HEADER/MOCK → 跳过   │
                                    │  数字 → 数据行         │
                                    └──────────┬────────────┘
                                               │
                                    ┌─ Cleanse（清理）────┐
                                    │  按 priority 排序    │
                                    │  执行列级/全局清理器  │
                                    └──────────┬───────────┘
                                               │
                                    ┌─ Validate（预校验）──┐
                                    │  Required (P=0)      │
                                    │  Length    (P=10)     │
                                    │  Pattern   (P=20)    │
                                    │  Dependency (P=40)   │
                                    │  任一失败 → 整行失败  │
                                    └──────────┬───────────┘
                                               │
                                    ┌─ Convert（导入转换）─┐
                                    │  ImportableConverter │
                                    │  按 priority 排序     │
                                    │  失败 = 校验失败     │
                                    │  含隐式校验：         │
                                    │  · Mapping找不到key   │
                                    │  · Enum匹配不到       │
                                    │  · 类型转换失败       │
                                    └──────────┬───────────┘
                                               │
                                    ┌─ PostValidate（后校验）┐
                                    │  Range (P=100)        │
                                    │  任一失败 → 整行失败   │
                                    └──────────┬────────────┘
                                               │
                                    ┌─ Construct（构建Bean）─┐
                                    │  反射实例化            │
                                    │  逐字段赋值            │
                                    └──────────┬────────────┘
                                               │
                                               ▼
                                    ImportResult<T>
                                    ├─ data: List<T>
                                    └─ failures: List<ImportFailure>
                                               │
                                    ┌─ GlobalValidate（全局校验）┐
                                    │  Unique 批次内唯一性检查    │
                                    │  从 successes 中移除重复行  │
                                    │  加入 failures              │
                                    └────────────────────────────┘
```

### 9.2 ImportPipeline 核心接口

```java
public class ImportPipeline<T> {

  public ImportResult<T> process(
    DataDefinition definition,
    Iterator<Map<String, String>> rowIterator,  // 流式行迭代器
    Class<T> beanType,
    String locale
  ) {
    // 1. 构建管线各阶段的执行器
    Map<String, ChainableCleanserExecutor> cleansers = buildCleansers(definition);
    Map<String, ChainableValidatorExecutor> preValidators = buildPreValidators(definition);
    Map<String, ChainableImportConverterExecutor> converters = buildConverters(definition);
    Map<String, ChainableValidatorExecutor> postValidators = buildPostValidators(definition);

    // 2. 逐行流式处理
    while (rowIterator.hasNext()) {
      Map<String, String> row = rowIterator.next();
      processRow(row, cleansers, preValidators, converters, postValidators);
    }

    // 3. 全局唯一性校验
    globalUniqueCheck();

    // 4. 构建结果
    return buildResult();
  }
}
```

## 十八、关键设计决策汇总

| # | 决策 | 结论 |
|---|------|------|
| 1 | 导入转换器是否保留 | 保留，独立设计 ImportableConverter 接口 |
| 2 | 转换器接口重构 | Converter→标记接口，ExportableConverter/ImportableConverter 分别定义方向性方法 |
| 3 | supports 方法 | 分为 supportsExport / supportsImport，语义各自独立 |
| 4 | 清理器是否独立 | 独立为 Cleanser 体系，@DataCleanser 注解 + SPI |
| 5 | 清理器与转换器复用注解 | 不复用，新建 @DataCleanser，概念清晰 |
| 6 | 转大/小写 | 作为 ImportableConverter 实现，非 Cleanser |
| 7 | 校验器扩展性 | @DataValidator 注解 + Validator 接口 + SPI，与转换器/清理器一致的扩展机制 |
| 8 | 校验分两阶段 | 预校验（转换前）+ 后校验（转换后） |
| 9 | 转换阶段的隐式校验 | MappingConverter/EnumConverter 转换失败 = 校验失败 |
| 10 | 严格模式 | 全局严格，任意字段失败→整行失败，不提供宽松模式 |
| 11 | Partial success | 不支持，一行要么全部成功要么全部失败 |
| 12 | 列匹配策略 | 代码标识行（Row 1）精确匹配字段名 |
| 13 | HEADER行和HINT行 | 合并为 Row 1（代码标识）+ Row 2（人类阅读+批注提示），共2行非数据行 |
| 14 | # 列双重语义 | 前2行=元数据标记(HEADER/MOCK)，数据行=序号(1,2,3...) |
| 15 | 模板颜色 | Row 1-2 浅蓝，Row 3(MOCK) 浅黄，数据区 白色 |
| 16 | 批注 | Row 1 每列加完整填写说明，Row 2 每列加格式提示（可选） |
| 17 | Excel 下拉验证 | mapping 列自动设置下拉列表 |
